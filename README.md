# 深读 Deepread

> 不是为了更快读完一本书，而是为了真正想清楚一件事。

Deepread 是一个本地运行的沉浸式 EPUB 阅读器。它把阅读流程从“打开书、划线、退出”改成一条更完整的思考链路：

```text
上传 EPUB -> 写下阅读意图 -> 沉浸阅读 -> 拖拽摘录 -> 写下自己的思考
-> AI 释义 / 反驳 / 联想 -> 完成回顾 -> 形成可导出的知识沉淀
```

它不是云端书库，也不是追求阅读速度的工具。它更适合用来读需要慢下来思考的书：产品、商业、哲学、社会学、心理学、技术思想类内容，或者任何你希望读完后能留下判断和行动线索的书。

## 应用价值

很多阅读器把重点放在管理书、同步书、快速翻页；Deepread 更关注读者的思考过程。

- 用“阅读意图”抵抗无目的的信息消费：每次打开一本新书前，先回答“为什么现在要读它”。
- 用“思维卡片”把划线变成观点：摘录只是原材料，真正有价值的是你当下的判断、疑问和联想。
- 用 AI 做思辨陪练：AI 不只解释摘录，也会结合你的批注和阅读目标，从释义、反驳、联想三个角度回应。
- 用完成回顾形成闭环：读到 95% 后，可以回看最初意图、沿途摘录和最终收获。
- 用 Markdown 导出带走结果：书摘和个人思考可以导出为 Markdown，继续放进笔记系统或知识库。

## 当前功能

### 书桌与书籍管理

- 上传本地 `.epub` 文件，书籍内容保存在浏览器 IndexedDB。
- 自动读取书名、作者，并尽量提取 EPUB 封面；失败时使用备用封面颜色。
- 书桌区分“正在阅读”和“知识沉淀”：进度达到完成状态的书会进入沉淀堆叠。
- 支持删除书籍；删除时会同时清理该书的内容和思维卡片。
- 无登录、无账号系统，默认面向本地桌面使用。

### 阅读体验

- 基于 epub.js 的连续滚动阅读模式。
- 自动保存阅读位置，下次打开回到上次停留处。
- 基于 EPUB CFI 计算真实阅读进度。
- 顶部显示书名、当前章节、进度百分比和累计阅读时长。
- 支持目录侧栏跳转。
- 支持正文搜索和卡片搜索；正文搜索结果可跳转并高亮关键词。

### 思维卡片

- 在正文中选中文字后，拖拽“抓取”按钮到右侧思维流，生成摘录卡片。
- 每张卡片包含书摘和“我的思考”。
- 卡片批注会自动保存，并在输入框失焦时立即落库。
- 支持删除单张卡片。
- 支持将当前书籍的全部卡片导出为 Markdown，包含书名、作者、阅读意图、摘录、个人思考和创建时间。

### AI 反思

每张思维卡片支持三种 AI 回应模式：

| 模式 | 作用 | 输入上下文 |
| --- | --- | --- |
| 释义 | 对摘录做深度解释和背景延展 | 书名、作者、阅读意图、书摘、你的思考 |
| 反驳 | 挑战观点盲区，提供反向视角 | 书名、作者、阅读意图、书摘、你的思考 |
| 联想 | 跨学科连接，生成新的理解路径 | 书名、作者、阅读意图、书摘、你的思考 |

AI 请求通过本地 Express 服务代理到 OpenAI 兼容接口，支持流式输出。环境变量仍使用 `MIMO_*` 命名，这是项目早期接入小米 MIMO 时留下的命名；实际可配置任意 OpenAI 兼容服务。

### 完成回顾

- 阅读进度达到 95% 后，可以打开“阅读完成回顾”。
- 回顾页展示最初阅读意图、摘录卡片预览和最终收获输入框。
- 保存回顾后，书籍进度会标记为完成。

## 快速开始

### 环境要求

- Node.js 18 或更高版本
- npm
- 一个 OpenAI 兼容的 AI API 服务

### 安装

```bash
git clone https://github.com/AyingAI/deepread.git
cd deepread
npm install
```

### 配置 AI

如果只阅读和做卡片，可以先不配置 AI；如果要使用释义、反驳、联想，需要配置 API Key。

方式一：在界面配置

1. 启动应用后进入书桌。
2. 点击右上角设置按钮。
3. 填写 API Key、API Base URL 和模型名称。
4. 可点击“获取模型”拉取可用模型列表。
5. 保存后配置会写入本地 `.env.local`。

方式二：手动创建 `.env.local`

```bash
cp .env.example .env.local
```

编辑 `.env.local`：

```bash
MIMO_API_KEY=your-api-key-here
MIMO_API_BASE=https://api.openai.com/v1
MIMO_MODEL=gpt-4o-mini
```

常见 OpenAI 兼容配置示例：

| 服务 | API Base URL | 模型示例 |
| --- | --- | --- |
| OpenAI | `https://api.openai.com/v1` | `gpt-4o-mini` |
| DeepSeek | `https://api.deepseek.com/v1` | `deepseek-chat` |
| 硅基流动 | `https://api.siliconflow.cn/v1` | `Qwen/Qwen2.5-7B-Instruct` |

注意：API Key 只保存在本地 `.env.local`，不要提交到 GitHub。

### 启动开发模式

```bash
npm run dev
```

打开浏览器访问：

```text
http://localhost:3000
```

进入页面后点击亮起的窗户，进入书桌。

### 生产构建

```bash
npm run build
npm start
```

生产模式下，Express 会直接提供 `dist` 中的前端静态文件。

## 使用方式

1. 进入书桌，上传一本 EPUB。
2. 首次打开书籍时，写下这次阅读的目的。
3. 在阅读页上下滚动阅读，应用会自动保存位置和进度。
4. 选中触动你的文字，把“抓取”按钮拖到右侧思维流。
5. 在卡片中写下你的思考。
6. 点击“解释”“反驳”或“联想”，让 AI 结合书摘、你的思考和阅读目标回应。
7. 读到 95% 后，打开完成回顾，写下最终收获。
8. 如需迁移笔记，点击右侧导出按钮生成 Markdown。

## 数据与隐私

- EPUB 文件、阅读位置、阅读意图、卡片和回顾内容保存在浏览器 IndexedDB。
- AI API Key 保存在项目根目录的 `.env.local`。
- AI 功能会把书摘、你的卡片思考、书名、作者和阅读意图发送给你配置的 AI 服务商。
- 默认服务监听 `0.0.0.0:3000`。如果你在局域网或服务器环境运行，请自行控制访问权限，避免把带 API Key 的本地服务暴露到公网。

## 技术栈

| 层 | 技术 |
| --- | --- |
| 前端 | React 19 + TypeScript + Vite |
| 服务端 | Express 5 |
| EPUB 渲染 | epub.js，通过 CDN 加载 |
| 动画 | framer-motion |
| 图标 | lucide-react |
| 样式 | Tailwind CSS，通过 CDN 加载 |
| 本地存储 | IndexedDB |
| AI | OpenAI 兼容 Chat Completions，支持流式 SSE |

## 项目结构

```text
deepread/
  index.html              # SPA 入口，加载 Tailwind / epub.js / 字体
  index.css               # EPUB iframe 样式
  index.tsx               # React 挂载入口
  App.tsx                 # 应用状态机：入口 -> 书桌 -> 阅读意图 -> 阅读
  server.ts               # Express 服务，AI 代理和设置 API
  types.ts                # Book / ThoughtCard 等类型定义
  utils/
    db.ts                 # IndexedDB 封装
  components/
    EntryView.tsx         # 入口仪式页
    DeskView.tsx          # 书桌、上传、删除、AI 配置
    RitualView.tsx        # 阅读意图仪式
    ReadingView.tsx       # EPUB 阅读器、搜索、卡片、AI、完成回顾
```

## 已知限制

- 当前主要面向桌面端，不针对移动端做适配。
- 当前只支持 EPUB，不支持 PDF、MOBI、AZW3 等格式。
- 数据保存在当前浏览器本地，暂不支持多设备同步。
- epub.js、Tailwind、字体和部分纹理资源依赖 CDN；完全离线环境下体验可能不完整。
- AI 质量取决于你配置的模型和服务商；不同 OpenAI 兼容服务对流式输出、模型列表接口的兼容程度可能不同。

## 路线图

- 优化 AI prompt，让 AI 更明确地优先回应“你的思考”，而不是只分析书摘。
- 补充更完整的阅读统计，例如书桌维度的阅读时长、卡片数量和完成趋势。
- 增加更强的卡片管理能力，例如排序、筛选、标签和批量导出。
- 改善离线能力，减少对 CDN 的依赖。
- 增加更稳健的导入 / 导出 / 备份机制。
